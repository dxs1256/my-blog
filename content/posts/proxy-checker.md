搞自动化的都懂——缺个干净活代理有多难受。网上那些免费代理榜单，别人机器上测得通，到你服务器上就不一定了，代理检测本质上是"你这台服务器能不能连上这个代理"的问题，没有统一答案。

最近 GitHub 上有个项目 **Proxy Checker** 挺火，就是解决这个的。

项目地址：https://github.com/strongshuai/proxy-checker

## 🎯 这玩意是干什么的

简单说，它是一个自托管的免费代理"体检中心"兼"蓄水池"。你往里塞持续更新的公共代理源，它全自动帮你测活、筛选，把真正能用的节点存起来。

![Proxy Checker 界面](https://i.ibb.co/Kc6p888h/24e770303de5.jpg)

## 🔌 支持哪些协议

通吃主流代理协议：

- **HTTP / HTTPS**
- **SOCKS4 / SOCKS5 / SOCKS5H**

没有协议前缀的代理会自动识别，覆盖很全面。

## 🧠 核心能力

### 通用代理检测

常规检测关注 HTTPS 基础连通、出口 IP、国家、IP 类型，结果会标记等级、稳定性和推荐用途。

### AI 服务专项检测

内置 5 种检测模式：

| 模式 | 检测目标 |
|------|---------|
| `generic` | 默认常规代理检测 |
| `openai` | 检测 ChatGPT / OpenAI API 可达性 |
| `claude` | 检测 Claude API 可达性 |
| `google` | 检测 Google 服务可达性 |
| `custom` | 自定义检测目标 |

这对于跑 AI API 的老哥特别实用——一个代理能不能用，直接针对性检测，不是笼统地测个通不通。

### 仓库管理

检测通过的好代理会进"我的仓库"，自动生成 **TXT 和 JSON 格式的直链**，你的脚本直接拉取就行。TXT 适合代理订阅，JSON 保留等级、国家、用途等完整信息。

支持按等级（A/B/C/D）筛选、复制、导出，还能清空指定等级。

## 🚀 部署方式

项目地址有完整部署说明，支持 Docker 和直接 Python 运行。部署在自己用代理的服务器上，这样检测结果才是真正对你有参考价值的。

```
谁要用代理，谁来测，结果才最有参考价值。
```

## 💡 说两句

免费代理这东西，网上资源不少但真能用的不多，手动测又累死人。Proxy Checker 把"拉取→检测→筛选→输出"整条链路自动化了，部署一次就能持续跑，省心很多。

尤其内置了 OpenAI、Claude 等 AI 服务的专项检测，对于经常折腾 API 的人来说挺实用。
