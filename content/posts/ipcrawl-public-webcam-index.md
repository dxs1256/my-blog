你有没有想过，你家门口那个摄像头，可能正在被千里之外的陌生人盯着看？这不是科幻电影里的桥段，而是正在发生的事实——全球有超过 13,000 个网络摄像头完全暴露在公网上，不需要任何密码就能访问。

IP Crawl 是一个公开网络摄像头索引系统，通过扫描公网上可直达的摄像头设备，汇集了来自全球的实时画面。办公室大堂、仓库车间、城市街道、工地现场、宠物店、码头泊位——目前索引了超过 13,700 个摄像头，全部无需密码即可查看。

项目地址:[https://ipcrawl.com](https://ipcrawl.com/)

---

🎯 **全球摄像头发现引擎**

IP Crawl 的工作方式其实很直接，但背后的数据量不小。定时爬虫从公开的互联网情报源中查询那些自带摄像头截图的设备，对每个结果只保留基本信息：大致位置（通过 MaxMind GeoIP 解析到城市级别）、托管运营商、和一张缓存截图。

这些摄像头不是通过漏洞入侵发现的——它们本来就暴露在公网上。大量网络摄像头厂商的出厂默认设置不开启任何访问控制，开箱即用就意味着任何人只要知道 IP 地址就能看。普通用户甚至不知道自己的设备能被外部访问，更不知道怎么改配置。

![IP Crawl 首页摄像头网格浏览](https://i.ibb.co/My36crTV/ipcrawl-home.png)

🎯 **摄像头浏览与多维度筛选**

首页以网格卡片形式展示所有摄像头，每个卡片包含实时画面（或定时快照）、地理位置、运营商信息、设备点赞数，以及当前状态标识——LIVE 表示设备在实时响应，SNAPSHOT 表示使用最近一次缓存截图。

筛选能力很全面：

按国家筛选：点开国家弹窗，可以看到每个国家有多少摄像头被收录。

按城市筛选：同一国家内进一步缩小范围，精确到城市级别。

按 ISP 筛选：不同运营商的设备暴露情况差异很大——有些 ISP 的摄像头特别多。

按制造商筛选：不同品牌设备的默认安全配置天差地别，这一维度对安全研究很有价值。

搜索框支持关键词联想，可以搜城市名、地标名、甚至是设备厂商型号。排序方式默认按 Most Favorited 排列，方便快速发现哪些画面最受社区关注。

🎯 **摄像头详情页：实时画面查看**

点击任意卡片进入详情页，可以看到完整的摄像头画面。画面有两种状态：

实时帧：设备在线且响应时，代理服务器直接向摄像头请求最新画面。

缓存快照：设备离线或无响应时，自动降级为最近一次成功抓取的截图。

详情页还列出了完整的设备信息——地理位置、托管 ISP、设备可能型号等。画面做了限速和缓存保护，不会对源设备造成负担。

![IP Crawl 摄像头详情页](https://i.ibb.co/FqhZLRBv/ipcrawl-camera.png)

🎯 **Am I Being Watched：查查你家附近**

这是 IP Crawl 最实用的功能之一。打开 /imce 页面，它会根据你的 IP 位置信息，显示附近是否有被索引的摄像头。

操作流程：打开 [https://ipcrawl.com/imce](https://ipcrawl.com/imce) → 页面自动检测位置 → 显示附近摄像头列表（如果有）。

如果发现自己家的摄像头出现在上面，说明这台设备正在被公网访问。解决方法是把摄像头从公网断开，或配置密码/WiFi 隔离。一旦设备不再对公网开放，IP Crawl 会在下一轮扫描中自动移除，不需要人工申报。

![Am I Being Watched 隐私检查功能](https://i.ibb.co/Cpr2jb46/ipcrawl-imce.png)

🎯 **统计面板：摄像头全景数据**

Stats 页面展示了 IP Crawl 收录摄像头的整体数据分布，包括：

按国家分布：哪些国家的摄像头暴露最多，排名和数量一目了然。

按 ISP 分布：哪些运营商的客户设备暴露最严重。

按制造商分布：哪些品牌的设备默认安全配置最成问题。

这些数据对物联网安全研究非常有价值——可以直观看到哪个厂商、哪个地区的问题最突出。

![IP Crawl 统计数据面板](https://i.ibb.co/7x9bbpD3/ipcrawl-stats.png)

🎯 **Console Mode：随机漫游玩法**

如果不想一个个翻看，Console Mode 提供了一个更像游戏的方式——模拟老式 CRT 显示终端的复古风格，随机把你丢到一个摄像头画面上。

包含功能：

随机频道：每次刷新跳转到一个随机摄像头。

猜地点游戏：根据画面猜地理位置，测试你的地理知识。

屏保模式：自动轮播摄像头画面。

可分享链接：把某个摄像头画面生成链接发给朋友。

适合纯粹出于好奇心的闲逛——你不知道下一个画面会是欧洲的街头、美国的仓库，还是亚洲的工地。

🎯 **关于隐私和安全的立场**

这是 IP Crawl 最受争议的部分，也是创始人 alec.is 在 About 页面中明确表态的地方。

Hacker News 上的讨论（48700834 号帖子）产生了明显分歧：反对者认为把摄像头整理成方便浏览的网站等于变相侵犯隐私，支持者认为曝光是推动改变的第一步。

IP Crawl 自己的立场很清晰——它希望通过可视化的方式，让摄像头和 IoT 行业的厂商对出厂安全负责。如果公众不知道问题的规模，厂商就没有动力改进。

去臃肿操作移除的组件无法从常规方式恢复，移除前看清每个项目的说明。建议先用 Minimal 预设熟悉功能。

⚠️ **注意事项**

摄像头移除方法：如果你的设备出现在列表里，从公网断开或配置密码访问即可。IP Crawl 的定时扫描会自动检测并移除，不需要提交申请。

地理位置精度：摄像头位置基于 MaxMind GeoIP 解析，精度只到城市级别。地图上的标记不是精确地址，不要试图用这个信息去定位具体房屋。

伦理边界：虽然这些摄像头本身是公开暴露的（不需要破解），但不代表它们的所有者愿意被公开浏览。浏览时请注意尊重。

关于设备安全：IP Crawl 本身不发起攻击、不尝试默认密码、不暴力破解。它只索引那些己经完全公开展示画面的设备。

状态标签含义：LIVE 表示当前能获取到实时画面，SNAPSHOT 表示使用缓存截图。SNAPSHOT 状态的设备可能在离线、网络波动或已更换配置。
