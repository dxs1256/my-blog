电脑里的视频文件越堆越多，NAS 硬盘告警、笔记本空间见底，想删又怕删错。更气的是同一个视频可能有多个版本——1080p 和 4K 各存了一份，水印版和无水印版同时留着，甚至不同剪辑片段散落在各个文件夹里。手动翻了一遍，眼睛花了也没找出几个重复的。

后来发现了 [**Video Duplicate Finder**](https://github.com/0x90d/videoduplicatefinder)（⭐3.4k），上面这些问题一次性全解决了。

项目地址：https://github.com/0x90d/videoduplicatefinder

## 🎯 这是什么

一个跨平台的视频/图片重复文件查找工具。和普通查重工具不同的是，它不依赖文件名或 MD5——它靠视频内容的**相似度**来判定重复。哪怕两个视频分辨率不同、帧率不同、甚至其中一个加了水印，它都能认出来。

C# 写的，跑在 Windows、Linux、macOS 上全没问题。AGPLv3 开源，802 次提交，一直在更新。

## 🎯 核心功能

### 跨分辨率/水印识别

这是它最值钱的地方。传统的 MD5 查重只能找到完全相同的文件，稍微转个码就认不出了。VDF 基于视频内容的视觉相似度做对比，能识别出：

- **不同分辨率**（720p vs 1080p）
- **不同帧率**（30fps vs 60fps）
- **带水印的版本**（加了台标/字幕的翻录版）
- **不同编码格式**（H.264 vs H.265）

每个匹配项会显示相似度百分比，你可以自己定阈值——低于多少不算重复，随你调。

### 局部片段检测

这个功能挺独特。当你有一段短片是从长视频里剪出来的（比如电影片段、直播精华剪辑），常规查重工具根本认不出。VDF 用音频指纹来做匹配——Chromaprint 风格的色谱提取 + 滑动窗口相似度匹配，定位到长视频里的具体位置，再用画面帧确认。

启用门槛也不高，在 Settings → Partial Clip Detection 里打开就行：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 最小片段/源比例 | 10% | 短于源视频 10% 的片段忽略 |
| 最小音频相似度 | 80% | 低于这个值的音频匹配被拒绝 |
| 要求视觉确认 | 开启 | 音画双重确认，减少误报 |
| 最小画面相似度 | 85% | 画面帧匹配阈值 |

### 三种操作界面

| 界面 | 适用场景 | 特点 |
|------|---------|------|
| **GUI 桌面版** | 个人电脑日常使用 | Avalonia 原生界面，操作直观 |
| **CLI 命令行** | 脚本自动化、定时任务 | 支持低质量/小文件/短时长等自动删策略 |
| **Web 网页版** | NAS、远程服务器、无头环境 | Docker 部署，浏览器访问 |

Docker 部署最简单：

```bash
docker run -d \
  --name vdf-web \
  -p 8080:8080 \
  -e VDF_WEB_PASSWORD=mypassword \
  -v vdf-db:/root/.config/VDF \
  -v vdf-state:/root/.local/state/VDF \
  -v /path/to/your/media:/media:ro \
  ghcr.io/0x90d/vdf-web:latest
```

打开 `http://localhost:8080` 就能用了。arm64 也支持，树莓派和 NAS 都能跑。

### 速度优化

- **FFmpeg 原生绑定**：不调外部进程，直接 C# 里调用 FFmpeg 解码，比走命令行快得多
- **快速重扫**：第一次扫描后会缓存哈希，第二次开始增量检查，只扫新文件

## 🎯 适合谁

- **NAS 用户**：硬盘空间金贵，几百 GB 的重复视频躺着吃空间
- **视频创作者**：剪辑素材越攒越多，导出几十个版本不知道哪些该删
- **收集党/仓鼠症**：下载的视频看到就存，从来没有去重
- **家庭媒体库维护者**：不同设备导入的视频可能有多个副本

## 总结

视频去重这个需求其实挺刚需的——只要你存视频，就迟早会遇到。VDF 把这件事做得够细：不同分辨率能认、水印版能认、片段剪辑也能认，三种操作界面覆盖了从个人桌面到 NAS 服务器的所有场景。部署也简单，一条 Docker 命令就完事。
